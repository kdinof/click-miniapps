## 1. Что это за проект

Кликабельный **frontend-прототип** (без бэкенда) десктоп-портала **Click MiniApps** —
мастер создания/онбординга мини-аппа. Собирается **1:1 по дизайну заказчика в Figma**.
Все экраны — это **состояния одного дашборда** (шаги мастера, переключение табов,
состояние токена). Язык интерфейса — **только RU**.

**Статус: готово полностью.** Собраны все 9 экранов-состояний + сквозной кликабельный
флоу. Прод-сборка проходит без ошибок. Каждое состояние сверено с макетами Figma
(скриншоты headless через Playwright → визуальное сравнение).

## 2. Стек и как запустить

- **React 19 + Vite 7 + TypeScript** (strict)
- **Tailwind CSS v4** — CSS-first, токены в `@theme` внутри `src/index.css`
  (отдельного `tailwind.config` нет — так работает v4 через `@tailwindcss/vite`)
- **lucide-react** — иконки
- Шрифт **Euclid Circular A** (OTF в `/fonts`, копии нужных начертаний в
  `/public/fonts`, подключены через `@font-face`)
- Состояние — **React `useReducer` + Context** (без react-router, это один дашборд)

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # tsc -b && vite build — должен проходить без ошибок
```

Node 25 / npm 11 на машине. Использовать **npm** (не pnpm).

## 5. Архитектура состояния

Весь стейт — в `src/state/dashboard.tsx` (reducer + context). Ключевое:

```ts
tab: 'dev' | 'contract' | 'info'
dev: 'initial' | 'tokenGenerated' | 'configuring' | 'configured'
contractSent: boolean
modal: null | 'success' | 'options'
toast: boolean
config: { subdomain: string; phones: string[] }
```

`App.tsx` рендерит каркас (Sidebar + светлая панель), заголовок, степпер (статусы
считаются из `getStatuses(state)`), табы, и контент активного таба. Модалы и тост —
поверх. Каждый `screens/*Tab` берёт state/dispatch из `useDashboard()`.

Переходы (экраны S1–S10 см. `SCREENS.md`):
- `GENERATE_TOKEN` → dev=tokenGenerated + toast (S1→S2+S3)
- `START_CONFIG` → dev=configuring (S3→S4)
- `LAUNCH` → modal=success (S4→S5)
- `CLOSE_SUCCESS` → modal=null, dev=configured (S5→S6)
- `EDIT_CONFIG` → dev=configuring (S6→S4)
- `OPEN_OPTIONS` → modal=options (S10)
- `SUBMIT_CONTRACT` → contractSent=true (S7→S8)
- `SET_TAB` → переключение табов

## 8. Как проверялась верстка

Браузерное расширение «Claude в Chrome» в этой сессии было **не подключено**, поэтому
для визуальной сверки использовался **Playwright headless** (chromium уже стоит в
`~/Library/Caches/ms-playwright`). Скрипт `/tmp/shot.cjs` прогоняет весь флоу и снимает
скриншоты в `.design-refs/render-*.png` (viewport 1440×2700). Их уменьшенные копии
(`*-sm.png`) сравнивались с макетами `.design-refs/s*.png`. Все 9 состояний совпали.

Для интерактивной проверки следующему агенту проще подключить Claude-in-Chrome или
открыть `http://localhost:5173` вручную.
